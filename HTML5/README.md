# Accessibility in Tech Writing and Design
## About Me
I am a senior at the University of Massachusetts Amherst studying professional writing and tech design and documentation. This project is part of the skills portfolio I have developed. It demonstrates my familiarity with several key software. 
## About the Project
This project was created in Madcap Flare and demonstrates my ability to use the tool and construct quality content with it. The content in this project includes an overview on the inclusion of accessibility features and philosophy while writing for tech design. It also contains research on various forms of accessibility.
### Team Content
This project was partially drafted with the help of various team members at the University of Massachusetts. The team content primarily focuses on transcription as an accessibility feature. It also includes information on transcription using the software ClipChamp.
- Meghan T: Overview of transcription and assistive technologies
- Kalana A: Guidance and best practices for transcription
### Tools and Concepts
This project began with research on different models of disability and then on different design philosphies regarding accessibility. A core concept in this project is **Universal Design**. 
>  "Universal design is the design of products and environments to be usable by all people, to the greatest extent possible, without the need for adaptation or specialized design."
-Ronald Mace
The project was designed in MadCap Flare. Flare is a content authoring tool with strong exporting capabilities, useful for organizing significant quantities of content. 
### Process
For the creation of team content, we organized the deliverable requirements into different sections. Based on our skill sets, each writer took responsibility for drafting content for each section and meeting overall output needs. We did individual research to draft content, compiled it into a single document, and conferred with each other to style the content and ensure all of the deliverables were met. 
## Takeaways
This project was an exercise in many new software tools and helped me develop skills, work with a team, improve my understanding on accessibility, learn about underappreciated design features, and more. Here are the top three things I learned from this project.
1. Ins and Outs of the Content Drafting Process
2. Powerful Tools for Content Authoring and Tech Design
3. Design Philosophies to Improve Access for All Users
